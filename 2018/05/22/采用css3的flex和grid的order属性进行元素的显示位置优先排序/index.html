采用css3的flex和grid的order属性进行元素的显示位置优先排序

Posted on 2018-05-22 Edited on 2020-10-11

通常在设计一个网站的时候，有的时候会有一些列表类的，也有图文混排的那种布局，PC版倒是没有什么，可是当我们在打开手机版的时候，发现，文字跟文字在一起，图片跟图片在一起了，显得不是很好看，这个时候，我们又不能对网站的布局进行修改，尤其是在wordpress中那种可视化编辑里面，直接左右布局的那种。

PC端类似这样的排版布局。当我们用手机打开的时候，就是这样的：

看到没，这个看起来体验就不是很好，我们更喜欢的应该是一行图片，下面是这个图片的说明文字，这个时候我们总不可能直接去网站的后台把这个东西再重新排版吧，即使重新排版将就了手机版，可是PC版又不是我们想要的样子，因此这种想法不是很好。

正解如下：

PC保持默认，手机版呢，我们对每个PC版的区块进行设定一个ID或者class，然后进行媒体查询语句，运用order这个属性。

好的，我们科普一下order这个属性：

order:用整数值来定义排列顺序，数值小的排在前面。可以为负值。

以下的css的写法：

order:-1;

order:0;

order:1;

order:2;

…

哪个对象的这个值越小，排序就越靠前。

因为PC版是采用两列式布局，假如代码结构如下：

<ul id="box" class="box">
    //A区
    <li class="a">A图片区</li>
    <li class="aa">A文字区</li>
</ul>

<ul id="box2" class="box">
    //B区
    <li class="b">B文字区</li>
    <li class="bb">B图片区</li>
</ul>

这里算是两个区块，手机版的话，就是呈现的是：A文字区跟B文字区靠在一起。不是很好看，我们需要做的是手机版是A图片区A文字区，B图片区B文字区；或者是B区不动，只需要把A区的文字区跟图片区重新排序即可。这个可是不改动整体内容的情况下进行的。

好的，我们的css代码就是：

方式一：B区不动，只需要把A区的文字区跟图片区重新排序即可

@media (max-width:767px){
 .box{
    display:grid;
}
.aa{
    order:-1 !important;
}
}

方式二：A区不动，只需要把B区的文字区跟图片区重新排序即可

@media (max-width:767px){
 .box{
    display:grid;
}
.bb{
    order:-1 !important;
}
}

记住：这个display:grid是在作用对象 (li) 的直属父级那里添加的，如果不是，可能达不到预期效果，比如以下的代码结构：

<div class="testa">
    <ul id="box" class="box">
        <li class="a">A图片区</li>
        <li class="aa">A文字区</li>
    </ul>
</div>

<div class="testb">
    <ul id="box2" class="box">
        <li class="b">B文字区</li>
        <li class="bb">B图片区</li>
    </ul>
</div>

如果以上的媒体查询代码：

@media (max-width:767px){
 .testa{
    display:grid;
}
.aa{
    order:-1 !important;
}
}

好的这个插曲到此结束，最后贴出最后的效果：

效果是不是好了很多？

如果这样写的话，可能A文字区不会把排序优先级往上提，也就是没有效果。这个笔者在处理一个手机版的样式美化的过程中实践过，写此文章仅仅用作笔记，如果针对这类问题有更好的建议，不妨加以斧正。
