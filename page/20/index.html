Magento模块中添加计划任务Cron Job

Posted on 2018-05-25 Edited on 2020-10-11

Magento提供了强大的计划任务功能(Cron Job)，使用计划任务，我们可以方便的按照一定的时间规律生成Google网站地图，更新汇率，清除日志，降价通知等等，此外我们还可以根据需要在自己开发的模块中添加计划任务的功能，下面我们将介绍怎样在自己的模块中添加计划任务。

首先我们需要开启Cron功能，在服务器上添加Cron，如果你用的cPanel面板，在面板中可以直接添加，如果是自己的主机，比如Linux主机，我们可以这样添加：

#crontab -e

添加

*/5 * * * * /bin/sh /var/www/magento1510/cron.sh

其中 */5 * * * * 表示系统将每5分钟执行一次这个脚本

/var/www/magento1510/cron.sh 是Magento网站根目录下的cron.sh文件的绝对地址，请根据自己的配置修改。

然后重启Cron

#service cron restart

Magento将每五分钟检查各模块下的计划任务，并生成计划任务表，保存在cron_schedule表中，当某个计划任务符合时间条件，Magento便执行该计划任务，并把执行的情况记录在该表中，下面我们看以下怎样在模块中添加计划任务。

一个典型的例子是newsletter，我们在/app/code/core/Mage/Newsletter/etc/config.xml中发现了下面一些配置：

<config>
    <crontab>
        <jobs>
            <newsletter_send_all>
                <schedule><cron_expr>*/5 * * * *</cron_expr></schedule>
                <run><model>newsletter/observer::scheduledSend</model></run>
            </newsletter_send_all>
        </jobs>
    </crontab>
</config>

上面的配置指定了一个计划任务newsletter_send_all，它每五分钟执行一次，将执行Mage_Newsletter_Model_Observer类中的scheduledSend()函数，该文件位于/app/code/Mage/Newsletter/Model/Observer.php，打开该文件就可以看到该计划任务执行的内容，如果我们要在自己的模块中添加计划任务，按照上面的例子在config.xml添加相应的contab即可。

当Magento产品属性或者是自定义属性为空时不显示该属性的值

Posted on 2018-05-25 Edited on 2020-10-11

当Magento中的产品的某个属性为空时，前台将显示为NO，如果需要隐藏此属性，可以按照以下方法做出修改：

打开 app/code/core/Mage/Catalog/Block/Product/View/Attributes.php

将 getAdditionalData 函数中

if (!$product->hasData($attribute->getAttributeCode())) {
    $value = Mage::helper('catalog')->__('N/A');
} elseif ((string)$value == '') {
    $value = Mage::helper('catalog')->__('No');
} elseif ($attribute->getFrontendInput() == 'price' && is_string($value)) {
    $value = Mage::app()->getStore()->convertPrice($value, true);
}

修改为

if (!$product->hasData($attribute->getAttributeCode())) {
    $value = '';
} elseif ((string)$value == '') {
    $value = '';
} elseif ($attribute->getFrontendInput() == 'price' && is_string($value)) {
    $value = Mage::app()->getStore()->convertPrice($value, true);
}

刷新前台产品页面，所有值为空的属性将不再显示。

获取Magento当前分类页的顶级父分类

Posted on 2018-05-25 Edited on 2020-10-11

有些时候，我们可能希望Magento获取到当前分类最顶层的分类。比如说，网站中包含一个类似如下的分类结构。无论是访问Apple还是3G的时候，我们都希望能获取到Cellphone这个顶级的父分类。

Cellphone/3G/Apple/

Magento默认有个getParentCategory()方法，该方法用于获取当前访问分类的直接父分类，如在访问apple的时候，使用该方法获取到的是3G，访问3G分类的时候，获取到的分类为Cellphone。

当我们希望获取顶层分类的时候，下面这段代码就可以实现。

$category = Mage::registry('current_category');
$it = 10;
if ($category) {
    while($category->getLevel() != 2 && $it > 0) {
        $category = $category->getParentCategory();
        if (!$category) {
           break;
        }
    }

    if ($category) {
        echo $category->getName();
    } else {
        echo 'Cannot find parent category';
    }
}

这是一个很简单的逻辑。希望获得更多的启发。

获取Magento购物车信息和当前网站店铺的信息，以及获取用户登录状态及登录用户信息

Posted on 2018-05-25 Edited on 2020-10-11

Magento 获取购物车信息常用代码

我们在做magento模板过程中，常常希望得到购物车的金额，货币信息等。

获取magento网店当前货币code：

echo $currency_code = Mage::app()->getStore()->getCurrentCurrencyCode();

获取magento网店当前货币符号信息：

echo Mage::app()->getLocale()->currency(Mage::app()->getStore()->getCurrentCurrencyCode())->getSymbol();

得到当前购物车中的全部产品的金额：

$grandTotal = Mage::getModel('checkout/cart')->getQuote()->getGrandTotal();

这样货币符号加上金额我们就得到了具体的当前金额了

得到当前购物的所在货币方式的全部金额

getQuote()->getGrandTotal();
getLocale()->currency(Mage::app()->getStore()->getCurrentCurrencyCode())->getSymbol().$grandTotal

在Magento购物车应用开发中，也经常需要判断当前用户的登录状态，并获取当前登录用户的相关信息。

首先，我们需要判断当前用户的登录状态，通过如下方式。如果用户已登录，则返回真，否则返回假。

Mage::getSingleton('customer/session')->isLoggedIn();

当判断用户已经登录的情况下，即可获取当前登录用户的相关信息。如用户名，姓名，邮件等。

$customer = Mage::getSingleton('customer/session')->getCustomer();
echo $customer->getName();
echo $customer->getEmail();

这里不仅可以通过getAttribute()的方式获取系统内置属性，如果你添加了一个customer的自定义属性，也可以通过此种方式获取该值。

$customer = Mage::getSingleton('customer/session')->getCustomer();
echo $customer->getFacebookUid();

店铺的信息获取方式

Magento的多店铺功能非常强大，通过多店铺功能，我们可以在一个Magento应用当中创建在外观上看来完全不相关联的多个店铺，并且可以绑定不同的域名。在Magento的开发当中，经常需要获取当前所在店铺的一些相关信息，例如店铺id，店铺名称及当前店铺所属Website的ID值。通过Magento的Mage类，我们可以获取到关于店铺的所有信息。

//获取店铺对象
<?php echo Mage::app()->getStore();?>

//获取当前店铺ID
<?php echo Mage::app()->getStore()->getStoreId();?>

//获取当前店铺Code，该Code在创建店铺时填写
<?php echo Mage::app()->getStore()->getCode();?>

//获取当前店铺所属的Website ID
<?php echo Mage::app()->getStore()->getWebsiteId();?>

//获取当前店铺的Name
<?php echo Mage::app()->getStore()->getName();?>

//获取当前店铺的状态
<?php echo Mage::app()->getStore()->getIsActive();?>

//获取当前店铺的URL
<?php echo Mage::app()->getStore()->getHomeUrl();?>

这样就得到当前购物车中的全部金额了！

采用css3的flex和grid的order属性进行元素的显示位置优先排序

Posted on 2018-05-22 Edited on 2020-10-11

通常在设计一个网站的时候，有的时候会有一些列表类的，也有图文混排的那种布局，PC版倒是没有什么，可是当我们在打开手机版的时候，发现，文字跟文字在一起，图片跟图片在一起了，显得不是很好看，这个时候，我们又不能对网站的布局进行修改，尤其是在wordpress中那种可视化编辑里面，直接左右布局的那种。

PC端类似这样的排版布局。当我们用手机打开的时候，就是这样的：

看到没，这个看起来体验就不是很好，我们更喜欢的应该是一行图片，下面是这个图片的说明文字，这个时候我们总不可能直接去网站的后台把这个东西再重新排版吧，即使重新排版将就了手机版，可是PC版又不是我们想要的样子，因此这种想法不是很好。

正解如下：

PC保持默认，手机版呢，我们对每个PC版的区块进行设定一个ID或者class，然后进行媒体查询语句，运用order这个属性。

好的，我们科普一下order这个属性：

order:用整数值来定义排列顺序，数值小的排在前面。可以为负值。

以下的css的写法：

order:-1;

order:0;

order:1;

order:2;

…

哪个对象的这个值越小，排序就越靠前。

因为PC版是采用两列式布局，假如代码结构如下：

<ul id="box" class="box">
    //A区
    <li class="a">A图片区</li>
    <li class="aa">A文字区</li>
</ul>

<ul id="box2" class="box">
    //B区
    <li class="b">B文字区</li>
    <li class="bb">B图片区</li>
</ul>

这里算是两个区块，手机版的话，就是呈现的是：A文字区跟B文字区靠在一起。不是很好看，我们需要做的是手机版是A图片区A文字区，B图片区B文字区；或者是B区不动，只需要把A区的文字区跟图片区重新排序即可。这个可是不改动整体内容的情况下进行的。

好的，我们的css代码就是：

方式一：B区不动，只需要把A区的文字区跟图片区重新排序即可

@media (max-width:767px){
 .box{
    display:grid;
}
.aa{
    order:-1 !important;
}
}

方式二：A区不动，只需要把B区的文字区跟图片区重新排序即可

@media (max-width:767px){
 .box{
    display:grid;
}
.bb{
    order:-1 !important;
}
}

记住：这个display:grid是在作用对象 (li) 的直属父级那里添加的，如果不是，可能达不到预期效果，比如以下的代码结构：

<div class="testa">
    <ul id="box" class="box">
        <li class="a">A图片区</li>
        <li class="aa">A文字区</li>
    </ul>
</div>

<div class="testb">
    <ul id="box2" class="box">
        <li class="b">B文字区</li>
        <li class="bb">B图片区</li>
    </ul>
</div>

如果以上的媒体查询代码：

@media (max-width:767px){
 .testa{
    display:grid;
}
.aa{
    order:-1 !important;
}
}

wordpress网站不同方式的彻底迁移注意细节

Posted on 2018-05-21 Edited on 2020-10-11

方式一

源码+数据库的迁移方式

优点：这种方式最彻底，最完整

缺点：风险较大，迁移成功则好；不成功的话，将会导致网站数据不完整，网站没法访问。

方式二

这个是通过后台数据导入导出的方式进行一个数据迁移。

优点：方便，简单，报错几率低。即使导入失败也不会危及网站的访问。

缺点：导入不彻底，媒体文件还需要重新上传到目标网站的upload中去。一些菜单和设置都不会被迁移，需要手动拷贝粘贴。工作量较大 。

备注 :在媒体文件从upload下载的时候，建议排查下，看下是否有多余的图片，因为wordpress会产生很多的缩略图，这些缩略图会占据很大的空间。这个是不得不考虑的因素。

笔者建议：如果是专业人士，建议采用方式一；如果不是专业人士，只是部分页面迁移的话，建议采用方式二。

Magento首页（子页）显示特价产品

Posted on 2018-05-17 Edited on 2020-10-11

1,建立文件app/code/core/Mage/Catalog/Block/Product/Special.php

class Mage_Catalog_Block_Product_Special extends Mage_Catalog_Block_Product_Abstract  
{  
    protected $_productsCount = null;  

    const DEFAULT_PRODUCTS_COUNT = 6;//默认显示数量  

    protected function _beforeToHtml()  
    {  
        $todayDate = Mage::app()->getLocale()->date()->toString(Varien_Date::DATETIME_INTERNAL_FORMAT);  

        $collection = Mage::getResourceModel('catalog/product_collection');  
        Mage::getSingleton('catalog/product_status')->addVisibleFilterToCollection($collection);  
        Mage::getSingleton('catalog/product_visibility')->addVisibleInCatalogFilterToCollection($collection);  

        $collection = $this->_addProductAttributesAndPrices($collection)  
        ->addStoreFilter()  
        ->addAttributeToFilter('special_from_date', array('date' => true, 'to' => $todayDate))  
        ->addAttributeToFilter('special_to_date', array('or'=> array(  
        0 => array('date' => true, 'from' => $todayDate),  
        1 => array('is' => new Zend_Db_Expr('null')))  
        ), 'left')  
        ->addAttributeToSort('special_from_date', 'desc')  
        ->setPageSize($this->getProductsCount())  
        ->setCurPage(1);  
        $this->setProductCollection($collection);  

        return parent::_beforeToHtml();  
    }  

    public function setProductsCount($count)  
    {  
        $this->_productsCount = $count;  
        return $this;  
    }  

    public function getProductsCount()  
    {  
        if (null === $this->_productsCount) {  
            $this->_productsCount = self::DEFAULT_PRODUCTS_COUNT;  
        }  
        return $this->_productsCount;  
    }  
}
