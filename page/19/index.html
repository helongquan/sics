通过jquery让没有链接的a标签不能被点击的多种方法

Posted on 2018-05-30 Edited on 2020-10-11

通过jquery让没有链接的a标签不能被点击的多种方法

比如：

这个东西，有很多文件可以点击下载，可是如果其中有个别的是空的链接，就是没有链接的资源的a标签，比如href="#"这种情况，如何高亮出来呢？然后又如何不让别人可以点击呢？

好的，这里我们采用jquery的方式解决这个问题。

第一，我们确保wordpress网站，或者是当前文件是引入了jquery这个库的。

然后，我们就是找到这个表格所在的a标签，由于这里笔者采用的是tablepress这个插件，所以通过开发者工具，我们定位到这个代码，然后添加jquery代码：

<script>

    $(function(){
    $(".tablepress tr td a[href='#']").addClass("disabled"); //添加a标签新的属性disabled
    $(".tablepress tr td a[href='#']").removeAttr('href');//去掉a标签中的href属性
    $(".tablepress tr td a[href='#']").removeAttr('onclick');//去掉a标签中的onclick事件
    });

</script>

以上的这句话的意思就是：我们找到这个表格中链接为#的a标签，动态添加一个class属性为：disabled。然后就是移除href这个属性。再然后移除onclick这个属性（这里防止a标签有这个属性，这个属性是可以调用javascript函数的），好的确保这两个被删除之后，链接就没法被点击了。

那么，事情还没完，我们还要高亮，这个就不得不说到这个动态添加的class这里了，好的对应的样式如下：

<style>
.tablepress tr td a.disabled .fa{
    color:#f39a00 !important;
}
</style>

这个就是a的class为disabled的下面的.fa的元素颜色为：#f39a00

好的，这样我们来到前台就可以看到效果了：

当我们鼠标飘到这个橘色的icon上面的时候，是不能点击的。大功告成！

magento性能优化，网站加速程序运行时间监测

Posted on 2018-05-29 Edited on 2020-10-11

在网站的优化过程中，通常需要查看程序的运行时间，mangento为我们提供了查看的方法。

这里用到的magento核心类是Varien_Profiler

用Varien_Profiler来监测程序运行时间的使用方法如下:

Varien_Profiler::start('self::app::init');    
self::$_app->init($code, $type, $options);//初始化APP  
Varien_Profiler::stop('self::app::init');

start和stop函数的参数是监测这段程序的名字，名字可以随便起，只要保证全局唯一就ok了.

这样在程序中写上代码后，监测的结果并不能直接显示，要显示结果需要有两个条件：

1、修改seystem/config/developer/调试(Debug)/配置器(Profiler) 为Yes

2、在index.php中去掉下面这段程序的注释

Varien_Profiler::enable();//启用Varien_Profiler

当上面这些都配置好后，刷新网页会在网页的底部显示如下的一个表格：

这样的表格是如何显示出来的呢，这里用到一个Block类/app/code/core/Mage/Core/Block/Profiler.php

class Mage_Core_Block_Profiler extends Mage_Core_Block_Abstract  
{  
    protected function _toHtml()  
    {  
        if (!$this->_beforeToHtml()  
            || !Mage::getStoreConfig('dev/debug/profiler')  
            || !Mage::helper('core')->isDevAllowed()) {  
            return '';  
        }  

        $timers = Varien_Profiler::getTimers();  

        #$out = '<div style="position:fixed;bottom:5px;right:5px;opacity:.1;background:white" onmouseover="this.style.opacity=1" onmouseout="this.style.opacity=.1">';  
        #$out = '<div style="opacity:.1" onmouseover="this.style.opacity=1" onmouseout="this.style.opacity=.1">';  
        $out = "<a href=\"javascript:void(0)\" onclick=\"$('profiler_section').style.display=$('profiler_section').style.display==''?'none':''\">[profiler]</a>";  
        $out .= '<div id="profiler_section" style="background:white; display:block">';  
        $out .= '<pre>Memory usage: real: '.memory_get_usage(true).', emalloc: '.memory_get_usage().'</pre>';  
        $out .= '<table border="1" cellspacing="0" cellpadding="2" style="width:auto">';  
        $out .= '<tr><th>Code Profiler</th><th>Time</th><th>Cnt</th><th>Emalloc</th><th>RealMem</th></tr>';  
        foreach ($timers as $name=>$timer) {  
            $sum = Varien_Profiler::fetch($name,'sum');  
            $count = Varien_Profiler::fetch($name,'count');  
            $realmem = Varien_Profiler::fetch($name,'realmem');  
            $emalloc = Varien_Profiler::fetch($name,'emalloc');  
            if ($sum<.0010 && $count<10 && $emalloc<10000) {  
                continue;  
            }  
            $out .= '<tr>'  
                .'<td align="left">'.$name.'</td>'  
                .'<td>'.number_format($sum,4).'</td>'  
                .'<td align="right">'.$count.'</td>'  
                .'<td align="right">'.number_format($emalloc).'</td>'  
                .'<td align="right">'.number_format($realmem).'</td>'  
                .'</tr>'  
            ;  
        }  
        $out .= '</table>';  
        $out .= '<pre>';  
        $out .= print_r(Varien_Profiler::getSqlProfiler(Mage::getSingleton('core/resource')->getConnection('core_write')), 1);  
        $out .= '</pre>';  
        $out .= '</div>';  

        return $out;  
    }  
}

wordpress运用jquery的全屏滚动插件fullpage实现鼠标滚动一次滑动一屏的效果

Posted on 2018-05-29 Edited on 2020-10-11

前一阵子，客户一直问，能够让wordpress实现一个鼠标滚动一次，页面滑动一屏，而不是一点点那种，就是全屏切换一下，还要确保菜单栏不被覆盖。

网上搜索了下，有这个效果，可是很多跟wordpress兼容起来不是很理想，也有类似插件，这里推荐一个：WP FullPage，但是这个插件有个不好的地方是，全屏可以实现了，但是网站的菜单栏没有了。

方案二：Scrollify

采用jquery的scrollify这个插件。

和fullPage.js 一样，Scrollify 也是一款基于 jQuery 的全屏滚动插件。跟 fullPage.js 相比，Scrollify 更加小巧，压缩后不足 4KB。但功能上不如 fullPage.js 强大，对移动设备的支持也不如 fullPage.js。不过对于一般的情况，Scrollify 是完全可以胜任的，它支持自定义锚链接、设置过度效果、偏移、是否显示滚动条、回调函数以及 fullPage.js 没有的局部滚动。

使用方法如下：

jQuery Scrollify

1、引入文件

<script src="js/jquery.min.js"></script>
<script src="js/jquery.scrollify.min.js"></script>

2、HTML

<div class="panel"></div>
<div class="panel"></div>
<div class="panel"></div>
<div class="panel"></div>

3、JavaScript

$(function() {
    $('.panel').css({'height': $(window).height()});
    $.scrollify({
        section: '.panel'
    });
});

描述：fullPage.js 是一个基于 jQuery 的插件，它能够很方便、很轻松的制作出全屏网站。

1、引入文件

<link rel="stylesheet" href="css/jquery.fullPage.css">

onSlideLeave某一水平滑块滚动前的回调函数，与 onLeave 类似，接收 anchorLink、index、slideIndex、direction 4个参数

缺点

由于wordpress很多的可视化编辑器都是动态调用的，所以会动态调用一些slider,动态调用产品的展示，以及别的一些东西，因为这些东西是实时更新变化的，写死在代码里面是不可取的，但是笔者尝试过运用简码的形式，把这些动态调用的东西，在wordpress里面封装成简码，然后调用。所以这个是没法用可视化的工具进行排版的，只能通过code的形式进行手动编码。不过稍微了解点前端三件套的人，应该没问题。

对自适应的挑战

可能是全屏滚动这种很大的一个原因是运用height:100vh，这个高度是屏幕全部高度，如果每一屏里面的东西很多，自适应的时候，内容难免会换行导致撑出一屏的高度，可能需要在手机上两屏才能显示完电脑端一屏的内容，这个时候就不行，因此想要做全屏滚动这种，内容尽量不要太多，特别是不要在里面放很多图片，如果你能够保证手机版也能显示没问题的话。

来自笔者的建议：

全屏滚动这种东西尽量放一个标题文字，一个说明，配合一个icon，然后来个按钮什么的即可。

magento中常见的js冲突，比如jquery冲突的解决办法

Magento内置使用的Prototype库，和jQuery的使用有一定的冲突，解决起来还是相当简单的。在网上有很多文章介绍这个东西，详细到说明了jquery和prototype库的引入前后顺序，最开始看的时候让我头晕而且很难记得，每次写的时候还要翻下文章。当然不是说那些文章不够好，只是个人感觉有点啰嗦。

这里记录下在Magento中使用jquery的具体方式。这种方式对于jquery与prototype库的先后引入关系没有限制。如下代码所示。可以看到，首先引入jquery库，接着引入需要的Jquery插件，然后就可以手写代码咯。首先调用jQuery.noConflict()更改jquery的命名空间，这样就可以使用jQuery替代$了。

<script src=”http://www.template.dev/js/jquery/jquery-1.7.1.min.js” type=”text/javascript”></script>
<script src=”http://www.template.dev/js/jquery/nivo/jquery.nivo.slider.pack.js” type=”text/javascript”></script>
<script type=”text/javascript”>
//<![CDATA[
jQuery.noConflict();
jQuery(window).load(function() {
jQuery(‘#slider’).nivoSlider();
});
//]]>
</script>

这里使用jQuery作为$的替代，稍微有点长了，如果使用$a,$b,$c的形式是不是更好呢？还有一种方式就是可以指定jQuery库的标识，和上面差不多，如下代码。

<script src=”http://www.template.dev/js/jquery/jquery-1.7.1.min.js” type=”text/javascript”></script>
<script src=”http://www.template.dev/js/jquery/nivo/jquery.nivo.slider.pack.js” type=”text/javascript”></script>
<script type=”text/javascript”>
//<![CDATA[
$j = jQuery.noConflict();
$j(window).load(function() {
$j(‘#slider’).nivoSlider();
});
//]]>
</script>

为Magento加速再做一点改进之移除不需要的js文件

Posted on 2018-05-26 Edited on 2020-10-11

Magento的速度一直是个让人头疼的问题，加大硬件投入是效果最明显的，当然程序层面还是要努力的优化。常见的优化方式就不多说了，不外乎开启gzip，关闭不需要的模块，装php加速器等等，网上有一篇已经被转载了无数遍的文章专门有讲到。今天讲到的是一些小手段，有多少效果还未知，不过至少不会变慢。

有些人可能会注意到，新安装的Magento项目，本身页面就载入了很多外置JS，这里以1.4.1.0为例，通过Firebug可以看到，在没有修改任何文件和添加任何数据的情况下，Magento首页就要载入13个外置JS，总计300K出头（不算Gzip压缩），可是Magento真的需要那么多JS吗，就算需要，那退一步讲，首页能用到的JS能有几个。首页的载入速度是网站速度中很重要的一环，一方面是因为这是用户看到的第一个页面，另一方面访问其他页面时，浏览器可以大量使用已经从首页读来的数据缓存，而不必重新完全载入，所以会比访问首页快得多。

所以我的方式就是想办法在首页去掉那些首页用不到的外置JS，减少请求数。经过不完全测试，针对我自己在做的项目，slider.js，dragdrop.js，builder.js和ccard.js对首页是无效的，移除后没有任何影响，同时据估计，如果首页用不到表单验证，validation.js应该也是没有用的（这个是大头，有34KB）。我并没有测试所有的JS，而且不同的项目有不同的首页，可能需求的JS也不同，只是提供一种思路，对页面无用的JS是没必要载入的。

如何只在首页不载入某些特定的JS，这里提供一种方式。我们知道，通用的JS文件是在layout目录下的page.xml中载入的，载入的方式如下:

<block type="page/html_head" name="head" as="head">  
                <action method="addJs"><mce:script type="text/javascript"><!--  
prototype/prototype.js  
// --></mce:script></action>  
                <action method="addJs" ifconfig="dev/js/deprecation"><mce:script type="text/javascript"><!--  
prototype/deprecation.js  
// --></mce:script></action>  
                <action method="addJs"><mce:script type="text/javascript"><!--  
lib/ccard.js  
// --></mce:script></action>

在head这个block中用的方式载入外部的JS文件，相对应的，我们同样可以用写xml的方式移除这些JS文件，方法如下

<reference name="head">
<action method="removeItem"><type>js</type><name>scriptaculous/builder.js</name></action>
<action method="removeItem"><type>js</type><name>scriptaculous/dragdrop.js</name></action>

因为只是在首页移除这些JS，其他页面还是要用的，所以把这段xml写在后台首页的CMS里。点击进入后台首页的CMS，点击左边的Design，把移除JS的代码写在Layout Update XML里面，保存后就可以看到效果，首页已经不会再载入你移除掉的JS，而其它页面依然存在。

同样的方式除了首页其他页面也可以使用，如果有足够的耐心的话，至少内容少了，访问速度总是会有所提升，不管效果明不明显。

PS：因为Magento的核心库prototype.js有127KB之巨，而绝大多数人应该是不会去动核心库的代码的，所以可以把prototype.js压缩，然后替换掉原来的，类似于jquery会提供一个jquery.min.js。自己去压缩一下prototype.js，通常是prototype.min.js

把prototype.min.js放置到js/prototype目录下，同时修改文件/app/design/frontend/default/yourtheme/layout/page.xml,
用

<action method="addJs"><script>prototype/prototype.min.js</script></action>

替换

<action method="addJs"><script>prototype/prototype.js</script></action>

大概可以为页面节省80KB的大小.

css3的keyframes如何让动画只执行一次就停下来，分别采用jquery和鼠标悬浮的方式处理

Posted on 2018-05-26 Edited on 2020-10-11

通常，我们在使用css3的动画特效，比如笔者最近就在帮助客户添加一个动画，左右摇摆的动画，动画加上去了，可是鼠标悬浮上去，或者鼠标不悬浮都一直在那里摇摆，看起来不太好，久了眩晕。

@-webkit-keyframes zy{

    transform: rotate(5deg);
  }
  40% {
    transform: rotate(-5deg);
  }
  50%,100% {
    transform: rotate(0deg);
  }

}

这个是左右摇摆的动画，于是我们确定某个区块(class=”goalelement”)运用它：

.goalelement{
  animation: zy 2.5s .15s linear infinite;
  -moz-animation: zy 2.5s .15s linear infinite; /* Firefox */
  -webkit-animation: zy 2.5s .15s linear infinite; /* Safari and Chrome */
  -o-animation: zy 2.5s .15s linear infinite; /* Opera */
}

这个一放上去就没完没了了，一直摇摆。

于是采用鼠标悬浮：

.goalelement:hover{
  animation: zy 2.5s .15s linear infinite;
  -moz-animation: zy 2.5s .15s linear infinite; /* Firefox */
  -webkit-animation: zy 2.5s .15s linear infinite; /* Safari and Chrome */
  -o-animation: zy 2.5s .15s linear infinite; /* Opera */
}

这样确实不会一直摇摆，可是当鼠标悬浮到这个区域的时候，还是一直摇摆，根本停不下来。

这个时候，我们需要采用一个新的东西（forwards），于是改成如下：

    animation: zy 3.5s .15s linear forwards;
    -moz-animation: zy 3.5s .15s linear forwards; /* Firefox */
    -webkit-animation: zy 3.5s .15s linear forwards; /* Safari and Chrome */
    -o-animation: zy 3.5s .15s linear forwards; /* Opera */

终于停下来了，这个需要给大家科普下：

zy是自己定义的动画帧，3.5s是整个动画的秒数，infinite是永久循环，如果只循环一次则采用：forwards，linear是线性变化

forwards：当动画完成后，保持最后一个属性值（在最后一个关键帧中定义）。

最后还有一个animation-direction 属性（alternate）定义是否应该轮流反向播放动画，默认情况下是normal。具体的可以参考W3C。

好的，这样的话，就算是停下来了。如果想要更加灵活点的话，我们笔者尝试着采用jquery动态添加样式。

<html>
<head>
<script src="http://libs.baidu.com/jquery/1.9.1/jquery.min.js"></script>
<style type="text/css">
    .zy{
        background:red !important;
        height: 200px !important;
        width:200px !important;
    }
</style>
<script type="text/javascript">
$(document).ready(function()
  {
  $(".noctiht-move").mouseover(function(){
    $(this).addClass("zy");
  });    
  $(".noctiht-move").mouseout(function(){
    $(this).removeClass("zy");
  });
});
</script>
</head>
<body>
<div class="noctiht-move" style="background:#98bf21;height:100px;width:100px;margin:6px;">
</body>
</html>

这样就可以把样式添加进去，覆盖掉原来的样式。同样的我们可以把动画写进预先准备好的class中，然后通过鼠标悬浮事件进行动态添加样式，然后根据鼠标离开这个区域额时候，动态的删除样式。

magento网站只有登录的用户才能看到网站的页面，未登录的用户一律跳转到登录页面！

Posted on 2018-05-26 Edited on 2020-10-11

我们经常看到很多网站每次访问的时候，都会重定向到网站的登录注册页面，这个时候，有些客户也有这个需求，只有登录的用户才能看到网站的页面，未登录的用户一律跳转到登录页面。

下面把自己做的步骤写下来和大家分享一下:

第一步：

在目录 /template/page/html 下新建一个文件，命令为：redirect.phtml

redirect.phtml的内容为：

<?php
Mage::getSingleton('customer/session')->setBeforeAuthUrl($this->getRequest()->getRequestUri());  //save requested URL for later redirection
if(!Mage::getSingleton('customer/session')->isLoggedIn()) {  // if not logged in
    header("Status: 301");
  // header('Location: '.Mage::helper('core/url')->getHomeUrl(customer/account/login)) ;  // send to the login page
    header('Location:'.Mage::getBaseUrl('web').'customer/account/login/') ;
    exit;
}

?>

笔者采用的模板目录：app/design/frontend/smartwave/porto/template/page/html/，这个大家根据自己模板（smartwave）目录自己寻找，porto是这个模板的主题名称。明确了这个之后，我们继续阅读。

第二步：

在page.xml写入代码，这个的目录在：app/design/frontend/smartwave/porto/layout/，打开page.xml文件

在文件page.xml中的

<default translate="label" module="page">
    <label>All Pages</label>
    <block type="page/html" name="root" output="toHtml" template="page/3columns.phtml">

的后面加上代码：

<block type="page/html" name="auth-redirect" as="auth-redirect" template="page/html/redirect.phtml"/>

上面这个是引用了我们前面新建的那个文件：redirect.phtml，并且给这个命了一个名字：auth-redirect

第三步：

在templatge/page的目录下（app/design/frontend/smartwave/porto/template/page）

1column.phtml

2columns-left.phtml

2columns-right.phtml

3columns.phtml

的这几个文件中的最前面加入代码

?>

好的，这个时候我们如果就开始访问的话，那么将会陷入一个死循环中，为啥昵？道理很简单，就是所有页面都是重定向，包括了用户的登录页面。这样引起的直接后果就是网站打不开，直至奔溃。
