注意：请操作前备份joomla全站，避免后续操作造成问题后无法恢复。

1.进入joomla管理后台，选择扩展-扩展管理-管理，然后选中需要删除的扩展，点击卸载；

2.上述操作无法彻底删除扩展时，手动删除joomla根目录下的以下目录以及相关名称文件；

/administrator/components/com_扩展名

/administrator/modules/mod_扩展名

/components/com_扩展名

/plugins/扩展名

/plugins/system/扩展名

/media/扩展名

3.删除数据库相关数据；

删除扩展名相关的表，例如数据库名为test，扩展为sh404sef，即删除test_sh404sef_XXX相关的所有表，

对应mysql命令如下：

mysql -u username -p

输入密码

>use test;

>show tables;

>drop table test_sh404sef_XXX;

删除test_extensions表、test_menu表中数据：

>select name from news_extensions;

>delete from test_extensions where name = ‘上条命令中查找到的所有与要删除扩展名相关的name’;

4.清除joomla缓存，网站-网站维护-清除缓存/清除过期缓存；

5.重新登录joomla后台，选择选择扩展-扩展管理-管理，已经找不到要删除的扩展，删除成功；

wordpress中针对jquery冲突造成的js特效解决方案

在wordpress主题当中，经常性会运用一些js插件或者特效，但是往往事与愿违，根据笔者的经历，遇到如下两个问题：

1、jquery被重复引用，直接导致的结果就是该页面中其他引用过jquery的效果显示不出来，比如revolution slider就会出现这种情况。针对这种情况只能取消引用jquery，这样有的时候对引进的特效不会产生影响，revolution slider也不会受影响。

2、另外一种情况，就是页面本身引用的jquery不能兼容引进的第三方特效，就是二者不能并存的这种情况，那么怎么办呢？由于直接修改jquery源代码比较难，那么我们可以采用iframe的方式。

这里介绍一下iframe的一些基本用法：

<iframe>是框架的一种形式，也比较常用到。

例子1

<iframe width=420 height=330 frameborder=0 scrolling=auto src=URL></iframe>

不用多说了。

width插入页的宽；height插入页的高；scrolling 是否显示页面滚动条（可选的参数为 auto、yes、no，如果省略这个参数，则默认为auto）；frameborder  边框大小；

注意：URL建议用绝对路径

传说中百DU用：<iframe width=“0” height=“0” frameborder=“0” scrolling=“auto” src=“WWW .webjx.com”></iframe>

例子2

如果一个页面里面有框架。。随便点页面里的连接，要求在这个<iframe> 里打开。在iframe 中加入name=** （**自己设定）

<iframe name=**  ></iframe>

然后在修改默认打开模式，：网页HEAD中加上<a href=URL target=**>或部分连接的目标框架设为（**）

例子3

要插入一个页面。要求只拿中间一部分。其他的都不要。

代码：

<iframe name=123  align=middle marginwidth=0 marginheight=0 vspace=-170 hspace=0 src="http://www.habotalk.com"  frameborder=no scrolling=no  width=776  height=2500></iframe>

控制插入页被框架覆盖的深度 marginwidth=0 marginheight=0；控制框架覆盖上部分的深度

vspace=-170

scrolling滚动条要否（auto、yes、no） frameborder框架的边框大小，width=776 height=2500此框架的大小。

wordpress后台中编辑模式下添加html代码的注意事项和心得分享

总结：在WordPress后台中添加html代码的注意事项和心得分享

作者很久前在接手一个网站需求的时候，客户的要求是用户在访问客户的网站的时候，一开始弹出一个层叠图，并且有提示（一些广告之类的东西）。

最开始我把这个功能写在一个html文件中，很显然，文件中包含了一些引用，比如jQuery以及自己的一些外部js代码，这个时候我直接把整个代码贴在了某个页面的对应的后台编辑器中，然后刷新前台，发现报错了，提示大概意思是你的这个文件里有jQuery，导致某些slider没法正常运转，后来我的解决思路是：

1、把这个html代码分成三部分，第一部分存放在一个html功能区块中

2、多次复用的html代码存放在另外一个html功能区块中

3、最后，把JavaScript代码和外部js文件存放在一个js功能区块中

注意：当然，可以根据需求可以分更多部分，不影响管理即可，最重要的是，这些被切分的区块必须符合一个完整的html文件的顺序，否则会出现意想不到的问题。

这里很值得提醒的是，因为很多的网站都会引用了jQuery，所以当我们把自己写好的引用了jQuery的html文件放进去的时候，很多时候会报错，原因是，jQuery被重复加载了，导致了一些错误。

解决方案是：把jQuery的引用去掉即可，因为整个网站中已经引用了jquery了，无须再次引用。

这里应该得到一些启发：一个完整的html文件是可以被分成多个片段放在不同的html功能区块中。当然，前提必须是在同一个页面，而不能跨页面存放。

WOW Slider如何去除水印整理

作为前端开发的兄弟们，接触频率较多的无非就那几种，作为现代网站开发中，轮播，旋转，鼠标悬停似乎变得必不可少，这些效果我们可以自己通过js的方法实现或者是css3来实现，当然这是对于比较熟悉代码的同学们而言的，如果对编程一点也不懂的同学估计有点难度；另一种方法，我们可以通过一些生成器或者生成器来完成网站的局部效果，这种方式较为通用，快速而高效，懂得软件的一般操作的从业人员基本就能掌握，废话说了很多，现在我来介绍一款软件：WOW Slider

这是一种幻灯片效果的生成器，操作容易，给我们带来了方便，我们再也不用费脑筋去一个个代码的敲出来了，同时有个美中不足的地方是，这款软件是收费的，而且价格不菲。当然，也有免费版的，但是都会在每一章图片后面自动添加了水印，让人很不舒服，那如何去除这个小小的logo呢？

我开始接触的时候，为了去除这个logo可谓费了不少时间，仍然找不到好的方法，百度搜索的都是如下的一些方案：

1、在wowslider.js里找到show()方法改为hide()，可能有几个show可以一个个试。

这个方法说得很含糊，因为wowslider.js有两个，一个是这个软件模板引擎的wowslider.js，另一个是生成幻灯片之后的在engine1文件夹里面的wowslider.js，这个会让那些初学者分不清楚，后来我根据这个提示两个都改了，貌似没啥用，被我放弃了。

2、在engine文件夹下编辑script.js，Ctrl+F：把var t=”xxxxxxxx”改成 var t=””

截图如下：

这种方式看起来挺详细的，这个是针对V8.2版本的，经过尝试由于版本不一样，没法实现，果断放弃了。而且对比了下这个版本跟最新版V8.7，界面太丑了，还是新版的好看，功能丰富了不少，所以没有使用旧版。

3、找到wowslider.js,打开之后搜索

("contextmenu",function(c){return false}).show()

把show()改成hide()就可以了

这种方法也被尝试了，还是不见好转，看来国内的解决方案有待提升呀，好多都是复制粘贴，转了一圈回来还是没找到。这个解决方案是来自这个网站：http://blog.wangjunfeng.com/archives/475

4、方法四

用firebug 定位 水印 灰色 字体 关键字：right: “5px” 发现在此修改css可将水印去掉

在wowslider.js中 搜索 5px 找到

 ai.append(ae) } K(ai).css({ zIndex: 11, right: "5px", bottom: "2px"}).appendTo(Q);

替换成 

ai.append(ae) } K(ai).css({ zIndex: 11, right: "5px", bottom: "2px", display: "none" }).appendTo(Q);

这种方法具有开创性，避开js,直接通过css的方法去掉这个水印，不过经过尝试，也以失败而告终，后来不知怎滴，我的天才般的想法诞生了。

我的方法是，继承方法四，我在想这个logo不是在右下角嘛，我就直接在那个div里面设置样式，直接，我在软件的模板引擎里面找到wowslider.js，找到

aI("<div>").css({ position: "relative", width: "100%", "font-size": 0, "line-height": 0, "max-height": "100%", overflow: "hidden"})

我之所以能找到这里也是通过开发者工具定位到样式，然后顺藤摸瓜找到这里，然后把上面的待改成

aI("<div>").css({ position: "relative", width: "100%", "font-size": 0, "line-height": 0, "max-height": "100%", overflow: "hidden","margin-bottom":"-30px" })

这样就直接把那个水印给遮住了，效果不受影响，后来测试了一下，发现兼容性有点问题，效果最好的是chrome和safari,有问题的是火狐和IE，那么怎么办呢？

我们还是回到开发这工具，通过设置幻灯片的高度来遮住这个logo即可。虽然这种方法很邪恶但是总比找不到方法又想用的了，再则咱们又不是用于商业用途，如果同学们想用于商业用途，建议还是要去买正版的，用着放心。
