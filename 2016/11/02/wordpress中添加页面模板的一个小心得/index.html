wordpress中添加页面模板的一个小心得

Posted on 2016-11-02 Edited on 2020-10-11

在前一段时间，根据客户的要求，解决一个文章分类页的排版问题，客户要求甚是奇葩，导致用了很多方法，都不是很满意，你可能会问，问什么会不满意，改个源代码不就可以了？说得很轻松，你知道的，从原则上讲，我们以不修改源代码的情况下来满足用户的需求为基本前提，修改源代码来达到用户的要求永远是最后的办法。

用过wordpress的同学就会知道，很多wordpress主题会通过shortcode调用post来调用文章显示在浏览器上，或者是直接通过主题的分类页模板（Template.php），调用文章显示在浏览器上，分类页大致情况就是缩略图+文章标题+摘要。

每次在后台写多一篇文章就会被渲染显示在分类页上，格式都会被固定了，想要修改排版布局，面临一个很大的风险就是可能会影响其他页面中使用过这个class的模块，为了达到这个要求怎么办呢？

鸢尾花序在这里想到了这么一个方法，就是单独添加一个跟以前一模一样的分类页面模板文件，我这里命名为（Template-new.php），然后修改这个模板里的模板名

<?php
/*
Template Name:鸢尾花序自定义模板
*/
?>

然后在Template-new.php 中找到需要改变的地方。由于我的问题是布局排版问题，于是我就在里面添加了一个div来包裹我需要修改布局的地方，为这个div添加一个ID，这样就可以直接对这个区块的各种元素通过css进行排版布局了，前提是文章分类页面必须在右边的模板那一个下拉框中选中“鸢尾花序自定义模板”这个模板，才能显示效果。

值得提醒的是：由于很多主题中这个分类页面模板代码逻辑复杂，如果自己设计得不那么完美，最好不要自己设计，直接在原来的代码中的div中添加class或者id，通过css来调整才是最好的方法。
