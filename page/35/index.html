如果是将一条横线贯穿那个文字的话，就好比删除线那样的效果具体代码如下：

 <style type="text/css">
       p{text-decoration:line-through;}
 </style>
 <p>test</p>

如果是文字在中间，文字两边有横线的话，实现方法如下：

html代码如下：

<div class="yuanweihua">
    <div class="yuanweihua-middle-line">
        <span class="yuanweihua-line-text">鸢尾花序列</span>
    </div>
</div>

css代码如下：

.yuanweihua{
    margin: 50px;
    text-align: center;
    background-color: #fff;
}

    content: '';
    display: block;
    height: 1px;
    width: 100%;
    background-color: #999;/*颜色需与主题大背景色一致，不然就露陷了*/
    position: relative;
    top: 10px;/*这个是用来调节线高*/

}
.yuanweihua-line-text{
    display: inline-block;
    background: #fff;
    padding: 0 18px 0 18px;
    position: relative;

首先为了数据安全，我们可以新建一个副表，来演示，然后把主表的字段中的数据导入到这个副表中，命令如下：

INSERT into school_list SELECT yuanxiao_name,logo,yuanxiao_area,yuanxiao_detail,constant from yuanxiao_details;

这个时候我们可以运用命令（这里以MySQL数据库为例）：

update 数据表名 set 表列名 = substring(表列名,截取的字符数)

我先运行这条命令：

update school_list set yuanxiao_area = SUBSTR(yuanxiao_area,5);

这个时候我们再用

select * from school_list

update school_list set constant = SUBSTR(constant,5);

update 表名 set 列名 = REPLACE (列名,"要去掉的字符","") where 列名 like "%要去掉的字符%" ;

我们还是以school_list这个表为例，我想删除所在城市中，有“市”的这个字符。

其实，如果想要删除某个字段后面的n个字符，规则是这样的：

update tablename set 列名 = left(列名,length(列名)-n);

这个时候我们需要明确下，这个数据表里面的文字是汉字还是英文，如果是英文的话，直接用length，如果是中文字的话，就改成char_length即可。

update school_list set yuanxiao_name = left(yuanxiao_name,char_length(yuanxiao_name)-2);

当前的路由：
destination 目的网段
mask 子网掩码
interface 到达该目的地的本路由器的出口ip
gateway 下一跳路由器入口的ip，路由器通过interface和gateway定义一调到下一个路由器的链路，通常情况下，interface和gateway是同一网段的
metric 跳数，该条路由记录的质量，一般情况下，如果有多条到达相同目的地的路由记录，路由器会采用metric值小的那条路由

第一条
缺省路由：意思就是说，当一个数据包的目的网段不在你的路由记录中，那么，你的路由器该把那个数据包发送到哪里！缺省路由的网关是由你的连接上的default gateway决定的
该路由记录的意思是：当我接收到一个数据包的目的网段不在我的路由记录中，我会将该数据包通过192.168.123.88这个接口发送到192.168.123.254这个地址，这个地址是下一个路由器的一个接口，这样这个数据包就可以交付给下一个路由器处理，与我无关。该路由记录的线路质量 1

第二条
缺省路由：该路由记录的意思是：当我接收到一个数据包的目的网段不在我的路由记录中，我会将该数据包通过192.168.123.68这个接口发送到192.168.123.254这个地址，这个地址是下一个路由器的一个接口，这样这个数据包就可以交付给下一个路由器处理，与我无关。该路由记录的线路质量 1

第三条
本地环路：127.0.0.0这个网段内所有地址都指向自己机器，如果收到这样一个数据，应该发向哪里 该路由记录的线路质量 1

第四条
直联网段的路由记录：当路由器收到发往直联网段的数据包时该如何处理，这种情况，路由记录的interface和gateway是同一个。
当我接收到一个数据包的目的网段是192.168.123.0时，我会将该数据包通过192.168.123.68这个接口直接发送出去，因为这个端口直接连接着192.168.123.0这个网段，该路由记录的线路质量 1

第五条
直联网段的路由记录
当我接收到一个数据包的目的网段是192.168.123.0时，我会将该数据包通过192.168.123.88这个接口直接发送出去，因为这个端口直接连接着192.168.123.0这个网段，该路由记录的线路质量 1

第六条
本地主机路由：当路由器收到发送给自己的数据包时将如何处理
当我接收到一个数据包的目的网段是192.168.123.68时，我会将该数据包收下，因为这个数据包时发送给我自己的，该路由记录的线路质量 1

第七条
本地主机路由：当路由器收到发送给自己的数据包时将如何处理
当我接收到一个数据包的目的网段是192.168.123.88时，我会将该数据包收下，因为这个数据包时发送给我自己的，该路由记录的线路质量 1

第八条

当我接收到广播数据包的目的网段是192.168.123.255时，我会将该数据从192.168.123.68接口以广播的形势发送出去，该路由记录的线路质量 1

大概就是这么用,如果你有两个网卡,或者两条线路,就可以用这个命令

Route

在本地 IP 路由表中显示和修改条目。

语法 

route [-f] [-p] [Command [Destination] [mask Netmask] [Gateway] [metric Metric]] [if Interface]]

参数 

-f

清除所有不是主路由（网掩码为 255.255.255.255 的路由）、环回网络路由（目标为 127.0.0.0，网掩码为 255.255.255.0 的路由）或多播路由（目标为 224.0.0.0，网掩码为 240.0.0.0 的路由）的条目的路由表。如果它与命令之一（例如 add、change 或 delete）结合使用，表会在运行命令之前清除。 

-p

与 add 命令共同使用时，指定路由被添加到注册表并在启动 TCP/IP 协议的时候初始化 IP 路由表。默认情况下，启动 TCP/IP 协议时不会保存添加的路由。与 print 命令一起使用时，则显示永久路由列表。所有其它的命令都忽略此参数。永久路由存储在注册表中的位置是 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\PersistentRoutes。

Command

指定要运行的命令。下表列出了有效的命令。 

命令 目的  
add 添加路由  
change 更改现存路由  
delete 删除路由  
print 打印路由Destination

指定路由的网络目标地址。目标地址可以是一个 IP 网络地址（其中网络地址的主机地址位设置为 0），对于主机路由是 IP 地址，对于默认路由是 0.0.0.0。

mask subnetmask
指定与网络目标地址相关联的网掩码（又称之为子网掩码）。子网掩码对于 IP 网络地址可以是一适当的子网掩码，对于主机路由是 255.255.255.255 ，对于默认路由是 0.0.0.0。如果忽略，则使用子网掩码 255.255.255.255。定义路由时由于目标地址和子网掩码之间的关系，目标地址不能比它对应的子网掩码更为详细。换句话说，如果子网掩码的一位是 0，则目标地址中的对应位就不能设置为 1。

Gateway

指定超过由网络目标和子网掩码定义的可达到的地址集的前一个或下一个跃点 IP 地址。对于本地连接的子网路由，网关地址是分配给连接子网接口的 IP 地址。对于要经过一个或多个路由器才可用到的远程路由，网关地址是一个分配给相邻路由器的、可直接达到的 IP 地址。

metric Metric

为路由指定所需跃点数的整数值（范围是 1 ~ 9999），它用来在路由表里的多个路由中选择与转发包中的目标地址最为匹配的路由。所选的路由具有最少的跃点数。跃点数能够反映跃点的数量、路径的速度、路径可靠性、路径吞吐量以及管理属性。

if Interface

指定目标可以到达的接口的接口索引。使用 route print 命令可以显示接口及其对应接口索引的列表。对于接口索引可以使用十进制或十六进制的值。对于十六进制值，要在十六进制数的前面加上 0x。忽略 if 参数时，接口由网关地址确定。
/?
在命令提示符显示帮助。
注释
路由表中 跃点数 一列的值较大是由于允许 TCP/IP 根据每个 LAN接口的 IP 地址、子网掩码和默认网关的配置自动确定路由表中路由的跃点数造成的。默认启动的自动确定接口跃点数确定了每个接口的速度，调整了每个接口的路由跃点数，因此最快接口所创建的路由具有最低的跃点数。要删除大跃点数，请在每个 LAN 连接的 TCP/IP 协议的高级属性中禁用自动确定接口跃点数。

如果在 systemroot\System32\Drivers\Etc 文件夹的本地网络文件中存在适当的条目，名称可以用于 Destination。只要名称可以通过“域名系统” (DNS) 查询这样的标准主机名解析技术分解为 IP 地址，就可以将其用于 Gateway，DNS 查询使用存储在 systemroot\System32\Drivers\Etc 文件夹下的本地主机文件和 NetBIOS 名称解析。

如果是 print 或 delete 命令，可以忽略 Gateway 参数，使用通配符来表示目标和网关。Destination 的值可以是由星号 () 指定的通配符。如果指定目标含有一个星号 () 或问号 (?)，它被看作是通配符，只打印或删除匹配的目标路由。星号代表任意一字符序列，问号代表任一字符。例如， 10.*.1, 192.168.、 127. 和 224 都是星号通配符的有效使用。

使用了无效的目标和子网掩码（网掩码）值的组合，会显示“Route:bad gateway address netmask”错误消息。目标中有一位或多位设置为 1，而其在子网掩码中的对应位设置为 0 时会发生这个错误。可以通过二进制表示法表示目标和子网掩码来检查这种情况。以二进制表示的子网掩码包括表示目标网络地址部分的一连串的 1 和表示目标主机地址部分的一连串的 0 两个部分。查看目标以确定目标的主机地址部分（由子网掩码所定义）是否有些位设置成了 1。

只有 Windows NT 4.0、Windows 2000、Windows Millennium Edition 和 Windows XP 的 route 命令支持 -p 参数。Windows 95 或 Windows 98 的 route 命令不支持该参数。

只有当网际协议 (TCP/IP) 协议在 网络连接中安装为网络适配器属性的组件时，该命令才可用。

范例

要显示 IP 路由表的完整内容，请键入： 

 route print

要显示 IP 路由表中以 10. 开始的路由，请键入： 

 route print 10.*

要添加默认网关地址为 192.168.12.1 的默认路由，请键入： 

 route add 0.0.0.0 mask 0.0.0.0 192.168.12.1

要添加目标为 10.41.0.0，子网掩码为 255.255.0.0，下一个跃点地址为 10.27.0.1 的路由，请键入： 

 route add 10.41.0.0 mask 255.255.0.0 10.27.0.1

要添加目标为 10.41.0.0，子网掩码为 255.255.0.0，下一个跃点地址为 10.27.0.1 的永久路由，请键入： 

 route -p add 10.41.0.0 mask 255.255.0.0 10.27.0.1

要添加目标为 10.41.0.0，子网掩码为 255.255.0.0，下一个跃点地址为 10.27.0.1，跃点数为 7 的路由，请键入： 

 route add 10.41.0.0 mask 255.255.0.0 10.27.0.1 metric 7

要添加目标为 10.41.0.0，子网掩码为 255.255.0.0，下一个跃点地址为 10.27.0.1，接口索引为 0x3 的路由，请键入： 

 route add 10.41.0.0 mask 255.255.0.0 10.27.0.1 if 0x3

要删除目标为 10.41.0.0，子网掩码为 255.255.0.0 的路由，请键入： 

 route delete 10.41.0.0 mask 255.255.0.0

要删除 IP 路由表中以 10. 开始的所有路由，请键入： 

 route delete 10.*

要将目标为 10.41.0.0，子网掩码为 255.255.0.0 的路由的下一个跃点地址由 10.27.0.1 更改为 10.27.0.25，请键入： 

 route change 10.41.0.0 mask 255.255.0.0 10.27.0.25

根据示意图：

B的有线网卡拿到数据包后看到目标网络是192.168.1.0，则把数据包交给路由表中指定的网关 192.168.1.1

192.168.1.1拿到数据包一看，哦，是给我的，打开一看，是192.168.0.3叫我，我要给它回话。

192.168.1.1是你的路由器，他只认识192.168.1.*的人，不认识192.168.0.*的人，所以路由器把数据包交给它的默认网关，他的默认网关是电信给的公网IP，公网IP拿到这个数据包发现目标地址是192.168.0.3，不认识，继续交给它的网关，它的网关也不认识，继续交给它的网关…..

之后这个数据包就在茫茫INTERNET中转手了255次后，被丢弃了。

所以0.3收不到回信的原因在路由器那里，只要在路由器那里再加一条路由，把目标网络192.168.0.0/24的数据包交给192.168.1.102来处理即可，192.168.1.102拿到回信后，发现目标网络是192.168.0.0/24，根据B的路由表，直接把回信给192.168.0.1来处理，刚好192.168.0.1是认识192.168.0.3的，所以回信由192.168.0.1交给了0.3。

这里我们来补充一下DNS的知识，DNS是域名解析系统，用来进行域名解析的。

当我们在一个局域中的时候，为了能够知道我们能否访问另外一台主机，我们通过ping的方式来进行测试，这个时候我们会用到IP，如果我们知道了IP我们就直接输入IP，很多时候，我们不容易记住IP，所以我们会记住域名就行了，这个时候如果把IP与域名进行关联起来的呢？这个时候就用到DNS了，先上几张图：

www.szu.edu.cn这个域名中，主机名是：www，域是：szu.edu.cn；也有可能碰到ftp.szu.edu.cn这样的地址，这个时候主机名是：ftp，域是：szu.edu.cn，不知道我们是否记得，我们在阿里云上购买了一个服务，然后把域名解析到对应的虚拟机上的时候，我们有个泛解析的问题，就是把www.xxx.com和xxx.com这样的访问效果一致，就是说可以不用输入www也可以访问到这个网站，这个就是*.xxx.com，就是说把所有是域为xxx.com这样的地址都解析到xxx.com来，主机名没限制。

打个比方，域名为www.szu.edu.cn对应的ip为210.33.22.44(对外，这个IP是电信提供的)，然后我们在这个域下面会有很多个主机名，比如zhangsan.szu.edu.cn这个对应zhangsan这个用户所对应的局域网的IP地址比如：192.168.1.110，就是说，在这个局域网中，李四（lishi）的ip是192.168.1.112，他所对应的域名是lishi.szu.edu.cn，那么李四如果想要访问张三的主机有两种方式，直接输入张三的ip(192.168.1.110)，另外一种方式是直接通过输入张三的域名（zhangsan.szu.edu.cn）访问，如果李四的主机没有设置DNS的话，那么是不能访问张三的主机的。

李四如何在自己主机上设置DNS呢？看截图：

这个DNS的地址设置为哪个才好呢？

我们知道，如果DNS没法解析是没用的，即使李四输入对了域名也不行，原因是DNS里面没有张三的信息，这个时候我们需要知道的是，李四和张三都是在同一个域里面，就是都属于szu.edu.cn这个域下面，都由这个管理，那么szu.edu.cn这个服务器里有这么一个DNS，这个里面就存了张三的域名和IP的关联消息，所以我们要把这个DNS地址设置为szu.edu.cn这台服务器的ip。

如果我们不知道的话，通过nslookup szu.edu.cn可以查看得到，或者是直接通过IP访问就不需要设置DNS了。

好了，刚才说的是局域网内这这么一个情况，那么比如李四想要访问百度怎么办？首先李四的主机已经设置好了DNS，而且DNS是szu.edu.cn这个的DNS，李四的主机是lishi.szu.edu.cn这个地址，百度是www.baidu.com，他们不再同一个网络，起码他们的域是不一样的，所以那么我们输入www.baidu.com的时候到底发生了什么情况。

首先李四的主机会访问所在的域（.szu.edu.cn）里面的DNS，发现里面没有一个叫做www.baidu.com的东西，然后就访问上一层域 .edu.cn，发现还是没有一个叫做www.baidu.com的地址，很显然，一个是.cn，另外一个是.com，所以肯定不认识，好了，然后继续往上一层域索引 .cn，这个时候还是没发现，于是继续走，来到了根目录.，这个时候所有顶级域名都只有一个根目录，所以这个时候找到了www.baidu.com，然后就把www.baidu.com对应的IP解析，然后把请求的服务返回给了李四。

从本质上来看，我们访问www.baidu.com，实际上还是访问的是163.177.151.109这个地址，只不过我们的DNS把www.baidu.com跟163.177.151.109这个域名相关联起来了。

眼尖的读者可能会发现，这里有一个非权威应答，这个的意思就是这个不是来自官方的响应，而是其他服务器保存了一些知名网站的副本缓存在了主机地址所在域的DNS，也可能是李四主机能够访问得到的DNS的缓存，一般这个缓存会保留5分钟更新一次，所以这个返回的消息只是一个缓存的内容而不是实时的目标地址返回的地址，缓存的大多是全球知名的服务商，旨在减少索引的层次，如果每次请求都要索引顶级的DNS，那么将会是效率很低的。

所以我们在能够上QQ而浏览器不能上网的话，很可能就是DNS出问题了找到一个能够访问的DNS设置在自己的主机即可。或者是我们直接输入访问的IP也可以上网。

DNS一般是在交换机上面已经有了的，或者我们家中的路由器。其实我们家庭用的那种路由器也是一台电脑，只不过它是一台微型电脑而已，有多个网卡（Lan接口），一个DNS服务器，DHCP服务器，就这些，可能有的路由器厂商服务不一样配置有所差异而已。

根据这个道理，我们完全可以把自己的家庭用的笔记本当做一个路由器来用，在笔记本上安装一个X-Router设置一下出口地址和入口地址，把笔记本安装多个网卡，一般我们的笔记本就两张网卡，我们也可以通过USB接口外设多张网卡，这样我们就可以把笔记本当做一个路由器来用了。

好，让我们回到虚拟机和笔记本用网线连起来，而虚拟机没法访问外网的问题，在这里面，如果虚拟机debian想要能够ping得通www.baidu.com的话，我们可以根据前面的路线图看到，它需要经过B的有线网卡，B的无线网卡，通过路由器，然后才能去外网，而B是通过X-Router这个连起来的，就等于是把B打通了，那么虚拟机debian必须配置DNS，不然是没法ping通的，要么是直接ping百度对应的IP。总之一句话：想要通过域名来访问目标，本机必须要设置能够访问得到目标地址的DNS，如果没有设置DNS的话，本机只能通过IP去访问目标。

计算机网络在VPN中的应用

Posted on 2017-05-08 Edited on 2020-10-11

在我们计算机网络中，会经常看到VPN这个东西，那么它是个什么东西呢？

我们来分析分析，VPN就是一个隧道模式，在互联网中，我们发送的消息是裸露的，没有经过加密，国内有针对国外的网络的防火墙，对访问的信息进行审核，层层过滤，所以我们在访问国外的一些信息的时候，比如一些敏感消息，是不能被访问的，那么我们在国内通过VPN这个工具来访问的话，是我们的访问信息先通过VPN服务器，比如VPN服务商在阿里云上购买了一个服务然后让我们的请求先进行一层加密，然后在经过国家的防火墙，国家防火墙对加密的信息没法识别，就没法拦截，于是我们的请求信息就可以通过了，当然也有可能政府防火墙那边设置了条件，一半通过一半不通过，所以我们有的时候会发现VPN不稳定，可能是政府那边的原因，也有可能是访问的服务器的原因。

像我们所居住的小区或者是生活区，会看到很多的牵网线的服务，他们其实也是先在电信那边购买网络服务（带宽足够大），然后通过虚拟化，远程客户端控制，切分成多个带宽。供各个用户使用，有的时候网络不稳定，也可能是电信那边对网络做了手脚，进行恶意竞争，导致网络不稳定。

从示意图可以看到，交换机是用来连接主机的（客户端），像机房里的电脑，都是通过网线连接到机房里的交换机（48端口的），而路由器是用来连接两个网络的，路由器与路由器的连接通过交换机连接。

交换机：用来连接主机或者是路由器的。

路由器：用来实现网络与网络之间的连接。

VPN服务商会在各个国家购买了服务器，然后把自己的服务存放在各个国家的服务器中，供用户访问各个国家，这就是我们所说的美国路线，日本路线，德国路线等等，都是这些情况。

要想理解两个网络的关系，可以通过两个村庄的关系来理解。

这个不得不说的是网关的问题，要想两台在不同网络中的主机相互认识，那么他们的通信是怎么个情况呢？

我们把网络A中的主机a，简称a，网络A简称A；我们把网络B中的主机b，简称b，网络B简称B.

A的路由器那里有一个网关，B的路由器那里也有一个网关，A和B之间有一个交换机，a把消息通过自身网络中的交换机把消息递交给网关，网关先检查这个请求的地址是否在同一个网络，如果不在同一个局域网里面的话，那么就直接递交给路由器，b也是一样，先把自己请求的消息经过所属的局域网的交换机然后来到b所在网络的网关，网关先检查一下请求的地址是否在同一个网络，如果不是，那么就交给B的路由器。这个时候路由器会有一个路由表，路由表会定期更新一次，路由器有多个网卡，分别连接多个网络，然后根据路由表来逐个排除，最终锁定请求的地址是属于哪个网络的，然后交给目标地址所属网络的网关，如果找不到目标地址所属网络的话，路由器会把消息反馈给主机，这样就能确定自己请求的消息是不是已经到达目的地了。

这个时候，你会很好奇，为啥能够认识网关呢？那是因为网关的ip地址是跟a或者b同属一个网络，这个网关既能识别A也能够识别B，所以他具有特殊的身份。

在不同两个网络里面通过IP就可以确认主机与主机之间是否同属一个网络，比如我们在同一个网络里面把其中一台主机的ip换了，这样就没法了连上网了，这样的话，我们把其中的两台主机都换成同一个网络号，那么他们之间是可以相互ping得通的，他们通过交换机进行相连，只是这两台修改网络号之后的主机跟其他的主机不在同一个网络里而已。
