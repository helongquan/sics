今日事今日毕，强迫症来袭，不写下来我会死

很久没碰过dede了，感觉有点不适应，最近接到一个任务，说要修改一些东西，前提条件是没有账户名，没有密码，有什么呢？有源代码，有提供服务器信息，数据库和数据库密码。

原主人为了保证网站安全，估计是做了一个安全策略，把默认的/dede文件夹，改成了其他文件，这样面对几十个文件夹，我找得颇费时间，终于找到了http://mywebsiteurl/xxx

这样，我顺利的进入到一个熟悉的界面：

可是，没有密码怎么办？这个时候，猜已经不是办法的情况下，我来了一个简单粗暴的方法：直接去数据库里面，找到存储这个登录信息的数据表，思路已经确定下来了，就开始行动，第一，找到网站的配置文件，跟其他网站有点不一样，不是config.php，而是common.inc.php（dede数据库配置文件所在路径为:/data/common.inc.php）。这样我就找到了这个文件，下载下来。里面的内容大致如下：

<?php

//数据库连接信息

$cfg_dbhost = 'HostName';

$cfg_dbname = 'DatabaseName';

$cfg_dbuser = 'DatabaseUsername';

$cfg_dbpwd = 'DatabasePassword';

$cfg_dbprefix = 'dede_';

$cfg_db_language = 'utf8';

?>

这样就可以知道这个网站跟哪个服务器以及服务器中的哪个数据库连接在一起，然后，获取数据库的登录地址，登录即可，这样我们就来到了数据表中，找到一个名叫：dede_admin的数据表，编辑，请看截图如下：

这样就可以找到登录名字和密码的，很不幸的是，这个密码被加密了，意思就是不能直接拿来就用，还得先初始化一下，根据上面的截图：

将“pwd”后面的这项值修改为f297a57a5a743894a0e4，这串数字就是织梦的默认密码admin，或者是改为c3949ba59abbe56e057f，这样初始密码就变成了123456。

点击运行，保存一下就行了。

这样，就成功的获得到了网站的登录密码和后台登录地址了。

值得提醒的是，当你用初始化的账号登录到dede后台后，不要忘了修改密码。为了安全。
