wordpress建立多站点的问题

Posted on 2017-07-07 Edited on 2020-10-11

我们都知道wordpress可以建立多站点，但是我们一不小心就可能选错了

建立子站点有两种选择，一种是子域名，另一种是子路径。

在.htaccess文件里配置为

RewriteEngine On
RewriteBase /
RewriteRule ^index\.php$ - [L]

# add a trailing slash to /wp-admin
RewriteRule ^([_0-9a-zA-Z-]+/)?wp-admin$ $1wp-admin/ [R=301,L]

RewriteCond %{REQUEST_FILENAME} -f [OR]
RewriteCond %{REQUEST_FILENAME} -d
RewriteRule ^ - [L]
RewriteRule ^([_0-9a-zA-Z-]+/)?(wp-(content|admin|includes).*) $2 [L]
RewriteRule ^([_0-9a-zA-Z-]+/)?(.*\.php)$ $2 [L]
RewriteRule . index.php [L]

的话就是子路径的，可能这里你还是很难区分，没关系，这里我们只需要打开wp-config.php文件的时候，找到配置多站点网络生成的那几句。上面的.htaccess里面的配置对应的wp-config.php的代码如下：

define('MULTISITE', true);
define('SUBDOMAIN_INSTALL', false);
define('DOMAIN_CURRENT_SITE', 'www.XXX.com');
define('PATH_CURRENT_SITE', '/');
define('SITE_ID_CURRENT_SITE', 1);
define('BLOG_ID_CURRENT_SITE', 1);

这里的配置为define('SUBDOMAIN_INSTALL', false);说明这个是不支持子域名的，那么就是支持子路径的。

而在.htaccess里面的配置为

RewriteEngine On

RewriteBase /

RewriteRule ^index.php$ - [L]

# uploaded files

RewriteRule ^files/(.+) wp-includes/ms-files.php?file=$1 [L]

RewriteCond %{REQUEST_FILENAME} -f [OR]

RewriteCond %{REQUEST_FILENAME} -d

RewriteRule ^ – [L]

RewriteRule. index.php [L]

的话，就是子域名的，这个文件所对应的wp-config.php的代码是

define('MULTISITE', true );

define('SUBDOMAIN_INSTALL', true);

$base = '/';

define('DOMAIN_CURRENT_SITE', 'iguitar.info');

define('PATH_CURRENT_SITE', '/');

define('SITE_ID_CURRENT_SITE', 1 );

define('BLOG_ID_CURRENT_SITE', 1 );

看哪里呢？看define('SUBDOMAIN_INSTALL', true);这里
